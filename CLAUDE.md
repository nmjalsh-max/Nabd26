# Nabd Space — HR Admin Dashboard

## نظرة عامة
لوحة تحكم لموارد بشرية (HR Admin MVP) لمنتج "Nabd Space"، تعرض:
- مؤشرات الأداء (KPIs): عدد الموظفين، نسبة التفاعل، متوسط الرفاهية النفسية، الجلسات القادمة.
- تبويب "Team wellbeing": جدول فرق قابل للبحث + لوحة تفاصيل جانبية.
- تبويب "Analytics": رسوم بيانية (Bar / Pie / Line) عبر recharts.
- تبويب "Programs": بطاقات جلسات (يوغا، ورش، استشارات).
- شاشة تحميل (Loading Screen) بتصميم قلب نابض SVG متحرك.

## المكدس التقني (Tech Stack)
- **Build tool:** Vite
- **Framework:** React 18 + TypeScript (JSX, بدون strict mode حالياً)
- **UI:** مزيج من inline styles (كائن `C` للألوان) + Tailwind utility classes (grid/flex فقط)
- **Charts:** recharts
- **Icons:** lucide-react
- **لا يوجد** router حالياً — التنقل بين التبويبات عبر state محلي (`useState`) في `App.tsx`

## هيكل الملفات
```
src/
  App.tsx      ← المكوّن الرئيسي (كل شيء حالياً في ملف واحد)
  main.tsx     ← نقطة الدخول (ReactDOM.createRoot)
  index.css    ← Tailwind directives
```

## قواعد الأسلوب (Coding Conventions)
- حافظ على نظام الألوان الموجود في كائن `C` بأعلى `App.tsx` — لا تُنشئ ألواناً hex جديدة متفرقة في الكود، أضِفها إلى `C`.
- المكوّنات صغيرة الحجم ومُقسّمة بتعليقات فاصلة على شكل `/* ── اسم القسم ── */` — حافظ على هذا الأسلوب عند إضافة أقسام جديدة.
- استخدم inline `style={{}}` للتنسيقات الدقيقة (ألوان، مسافات محسوبة)، واستخدم Tailwind classes فقط للتخطيط العام (`flex`, `grid`, `gap`).
- البيانات التجريبية (mock data) موجودة أعلى `App.tsx` (`teams`, `sessions`, `alerts`, ...) — عند الربط بـ API حقيقي، استبدلها بـ hooks (`useEffect` + `fetch`) مع الحفاظ على نفس الأسماء والأشكال (shape) قدر الإمكان لتقليل التعديلات على المكوّنات.
- الخطوط: Inter (نص عام)، Sora (عناوين)، JetBrains Mono (أرقام/قيم).

## أهداف التحسين القادمة (اقتراحات للعمل عليها)
1. تقسيم `App.tsx` إلى ملفات منفصلة تحت `src/components/` و `src/data/` و `src/types.ts`.
2. إضافة React Router لدعم روابط مباشرة لكل تبويب.
3. ربط البيانات الحقيقية عبر API بدلاً من mock data.
4. إضافة اختبارات (Vitest + React Testing Library).
5. تحسين إمكانية الوصول (a11y): aria-labels، تباين الألوان، دعم لوحة المفاتيح.
6. دعم اللغة العربية (RTL) بشكل كامل إن كان الجمهور المستهدف عربي.

## قواعد عامة للمساعد (Claude Code)
- لا تُثبّت مكتبات جديدة دون ذكر ذلك صراحة في الرد.
- حافظ على التصميم البصري الحالي (الألوان الداكنة، الـ glow effects) إلا إذا طُلب تغييره.
- عند إعادة الهيكلة (refactor)، حافظ على نفس الأسماء المُصدَّرة (exports) قدر الإمكان حتى لا تنكسر الاستيرادات.
